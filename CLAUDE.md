# CLAUDE.md — readmission-risk-scorer

## Project Identity
Classical ML prototype predicting 30-day hospital readmission risk
at discharge using XGBoost + SHAP. Provider-side use case.
No LLM in the critical inference path. Pre-GenAI technique stack
demonstrating full ML spectrum alongside the GenAI prototypes.
Addresses CMS HRRP penalty context: up to 3% Medicare DRG payment
reduction for excess readmissions.

## Stack
- Runtime: Python 3.11 (uv)
- ML: XGBoost, scikit-learn (StandardScaler, GridSearchCV,
  calibration, metrics), SHAP (TreeExplainer)
- Inference API: FastAPI + uvicorn
- Validation: Pydantic v2
- Visualization: matplotlib, seaborn (eval plots)
- Testing: pytest + httpx
- IaC: Terraform (Azure ML + Functions + APIM + Key Vault)
- Optional LLM: Anthropic Claude via anthropic SDK
  (Track 2 only; /predict/explain endpoint)

## Critical Constraints
- No real PHI anywhere; all patient records are synthetic
- MOCK_MODEL=true for all tests; never load model or call
  external API in test suite
- Trained model artifacts committed to models/; do not delete
- reports/ committed after training run; do not regenerate
  without re-running train.py + evaluate.py
- Pricing constants in src/utils/cost.py must be dated
- OOD check and imputation must be tested independently

## Architecture Principles
XGBoost is the right tool for this problem: tabular features,
< 50K samples, interpretability requirement, < 5ms latency
requirement. SHAP is not optional explainability; it is the
trust infrastructure that makes the prediction usable. The
LLM narrative layer (/predict/explain) is additive and never
in the critical path.

Categorical features use ordinal encoding with fixed mappings defined in
src/utils/inference.py (ENCODING_MAPS). This must be identical between
generate-dataset.py, train.py, and the inference route.

## Autonomous Build Rule
Work through the build sequence phase by phase without stopping to ask
for confirmation between steps, unless you hit a Manual Execution Gate
(see below). Complete each phase fully before starting the next. When a
phase produces a runnable artifact, confirm it runs before moving on.
Run uv sync, pytest (MOCK_MODEL=true), and uvicorn smoke tests
autonomously. Never attempt to run scripts/generate-dataset.py,
scripts/train.py, or scripts/evaluate.py autonomously; these are
manual execution gates.

## Manual Execution Gates
These steps require Paul to run a command manually. When you reach one,
stop, print the exact command and expected output, and wait for Paul to
confirm before proceeding to the next phase.

Gate 1 — end of Phase 2 (after generate-dataset.py is written):
  STOP. Run this command:
    uv run python scripts/generate-dataset.py
  Expected: data/train.csv (4,000 rows) and data/test.csv (1,000 rows)
  Tell Paul: "Gate 1 reached. Run the command above and confirm
  train.csv and test.csv exist before I continue to Phase 3."

Gate 2 — end of Phase 3a (after train.py is written):
  STOP. Run this command:
    uv run python scripts/train.py
  Expected: models/xgb_readmission_v1.joblib, models/scaler_v1.joblib,
            models/feature_names.json (~2-5 minutes for GridSearchCV)
  Tell Paul: "Gate 2 reached. Run the command above and confirm model
  files exist in models/ before I continue to Phase 3b."

Gate 3 — end of Phase 3b (after evaluate.py is written):
  STOP. Run this command:
    uv run python scripts/evaluate.py
  Expected: reports/eval-report.json, reports/shap-summary.png,
            reports/roc-curve.png, reports/calibration-plot.png,
            reports/training-cost.json
  Tell Paul: "Gate 3 reached. Run the command above and confirm
  all five report files exist in reports/ before I continue to Phase 4."

After Paul confirms each gate, continue to the next phase automatically.
